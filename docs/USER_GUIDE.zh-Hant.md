# Zomate Fitness 使用手冊（繁體中文）

本文件說明 **Zomate PT 管理系統** 的主要功能與日常操作流程，適合管理員、櫃台人員與教練參考。

---

## 一、系統簡介

本系統用於健身中心管理：**學員入會與文件**、**課程／簽到**、**WhatsApp 相關流程（介面與模擬）**，以及**財務與報表**。

| 區塊 | 說明 |
|------|------|
| 前端 | Next.js（App Router）、TypeScript |
| 後端 API | 可連線至 Kotlin Spring Boot；本機未設定 `NEXT_PUBLIC_API_BASE_URL` 時，會由 Next.js Route Handlers 模擬 API |
| 詳細技術與路線圖 | 請參考總庫或內部的 `docs/REQUIREMENTS.md`；模擬 API 行為見 `docs/MOCK_API_SIMULATION.md`（若本倉庫未附完整檔案，請向開發團隊索取） |

---

## 二、後台登入

1. 開啟瀏覽器，進入應用程式的 **登入頁**（路徑：`/login`）。
2. 輸入帳號、密碼後送出。
3. 登入成功後會進入後台（預設導向 `/admin`）。

**內建示範帳號（開發／示範環境）：**

- **管理員（ADMIN）**：`masterzoe` / `12345678`
- **櫃台（CLERK）**：`worker` / `12347890`

登入狀態會儲存在瀏覽器的 **localStorage**。若需換帳號，請先登出再重新登入。

## 三、後台介面導覽

登入後會進入共用後台版型（左側選單＋頂欄）。側欄會定期檢查 **API 連線**；若使用本機模擬 API，資料庫健康檢查可能顯示為不適用（NA）。

### 3.1 Dashboard（後台面板）

- **後台面板**（`/admin`）：總覽與快捷入口。

### 3.2 學員管理（Student Management）

- **學生名單**（`/admin/students`）：檢視與管理學員資料。
- **入職紀錄／健康表單**（`/admin/onboarding-records`）：與入會、PAR-Q 等表單紀錄相關之查閱與管理。

### 3.3 課程與出勤（Course & Attendance）

- **教練日程·簽到**（`/coach/calendar`）：教練端日程與簽到相關畫面。
- **教練課表**（`/coach`）：課表檢視。
- **QR 簽到中心**（`/admin/attendance/qr-console`）：管理／產出 QR 簽到相關流程之控制台。
- **Session Ledger**（`/admin/attendance/session-ledger`）：節數／堂數帳本；可記錄理由類型（例如：實際出席、遲到取消、教練補堂等）。報表支援匯出 Excel。
- **試堂／開課管理**（`/student/trial`）：試聽課程相關管理（與完整後端整合狀態請以專案需求文件為準）。
- **學生入口**（`/student`）：學員面向之入口頁連結。

### 3.4 財務與行政（Finance & Admin）

- **銷售與分期**（`/admin/finance/sales`）：月銷售報表、分期欄位等；可匯出 Excel。
- **支出管理**（`/admin/finance/expenses`）：支出／單據類資料；可新增與匯出。
- **薪酬／出勤輔表**（`/admin/finance/payroll`）：會籍與教練出勤相關報表；可匯出 Excel。

### 3.5 系統設定（System Settings）

- **WhatsApp API 狀態**（`/admin/settings/whatsapp`）：檢視 WhatsApp 整合相關狀態與設定介面。
- **免責聲明內容設定**（`/admin/settings/disclaimer`）：維護免責聲明等文案設定。

## 四、學員端常用流程

以下為學員或現場使用之公開／半公開路徑（無需後台帳號即可完成部分步驟，實際依部署設定為準）。

### 4.1 新學員入會與文件（Onboarding）

- 路徑：**`/student/onboard`**
- 依精靈步驟填寫：姓名、電話、電郵、**香港身份證（HKID）** 等。
- **PAR-Q** 七題為是／否；若有任一是「是」，需配合 **醫療 clearance**（系統會要求記錄／上傳相關資訊，詳見畫面）。
- 可區分**新入會**與**續會**，並可填寫備註。
- 政策提示：系統會帶出會籍效期邏輯（例如 **10 堂對應 3 個月**、**30 堂對應 6 個月**）與 **7 天冷靜期** 確認（以畫面與後端實作為準）。

### 4.2 QR 簽到（Check-in）

- 路徑：**`/student/checkin`**
- 可使用 **掃描 QR** 或 **貼上 QR 內容** 完成簽到。
- 另有 **PIN** 驗證及選填 **電話** 後備流程。
- 實務上可能僅允許 **當日** 簽到（依專案需求與實作）。

### 4.3 學員首頁

- 路徑：**`/student`**  
  作為學員功能入口或指引頁。

## 五、四大核心業務（概念摘要）

1. **數碼化行政入職**：學生填表與免責／健康聲明 → 後台可即時查閱紀錄。  
2. **試堂與購買**：試堂註冊、收款確認後，可透過 WhatsApp 等管道通知餘額（依實際整合為準）。  
3. **智能 QR 簽到**：掃碼與 PIN → 扣堂與相關通知（依實際整合為準）。  
4. **管理後台財務**：入機、分類統計、月報與 Excel 匯出。

## 七、疑難排解（簡要）

| 狀況 | 建議 |
|------|------|
| 無法登入後台 | 確認帳密、清除本網站資料後重試，或確認後端／模擬 API 是否啟動。 |
| 資料與預期不符 | 若未設定 `NEXT_PUBLIC_API_BASE_URL`，目前可能為**記憶體模擬資料**；正式環境請連線真實 API。 |
| 匯出 Excel 異常 | 確認瀏覽器未阻擋下載，並確認該頁資料已載入完成。 |
